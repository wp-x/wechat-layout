# wechat-layout

把文章丢给 AI，拿回一份能直接粘贴进公众号编辑器的排版 HTML。

它的本体是几份 Markdown 写的排版规则，哪个模型都能读。ChatGPT、Gemini、DeepSeek、Kimi 的网页对话里能用，Claude Code、Codex、Cursor 这类编码工具里也能用。不装任何东西的用法：把 `SKILL.md` 和 `references/` 里的文件内容拼在一起，加上你的文章，发给模型。

## 为什么做这个

公众号编辑器不支持 Markdown，第三方排版工具要一段一段手动刷样式。我想要的流程是写完文章，说一句"排成公众号格式"，然后复制、粘贴、发布。

市面上已有类似的排版提示词，但大多只有一套固定配色，样式写死在提示词里，换个风格就得重写整份。这个仓库把样式拆成三个互不干扰的维度，组件只写一遍，靠替换配色令牌和形态规则来组合。

## 三个维度

**8 套配色主题**（`references/themes.md`）
清新绿、科技蓝、商务黑金、活力橙、优雅紫、中国红、少女粉、极简黑白。每套是一张 10 个值的令牌表，也支持从你的品牌色派生一整套。

**9 种设计风格**（`references/styles.md`）
柔和卡片风（默认）、新粗野主义、蒸汽波、瑞士国际主义、国风雅致、复古终端、孟菲斯、杂志编辑风、渐变玻璃感。风格改的是形态——边框、阴影、圆角、字体、分隔符造型，和配色正交。新粗野主义是 3px 黑描边加硬投影，蒸汽波是粉紫青渐变加全角字符，复古终端是深底终端绿加 `$` 前缀标题。

**6 种文章预设**（`references/presets.md`）
干货教程、观点评论、情感故事、资讯速报、商务通告、深度长文。决定组件搭配的节奏，比如情感故事少用卡片、资讯速报段距收紧。

组件在 `references/components.md`，24 个：标题区块、金句卡、步骤时间线、对比双栏、FAQ、数字亮点、引导关注卡这些，颜色全部用 `{{令牌}}` 占位。

## 怎么用

### 网页对话（ChatGPT / Gemini / DeepSeek / Kimi / Claude）

把下面四个文件的内容拼成一条消息发过去，文章跟在后面：

```
SKILL.md
references/themes.md
references/styles.md
references/components.md

现在请排版这篇文章：<正文>
```

嫌长的话，只带 `SKILL.md` + `themes.md` 里你要的那一套主题 + `components.md`，也够用。

### Claude Code

```bash
git clone https://github.com/wp-x/wechat-layout.git ~/.claude/skills/wechat-layout
```

之后对话里说"帮我排版这篇文章"就会自动触发。

### Codex / Cursor / 其他编码 Agent

把仓库 clone 到项目里，在 AGENTS.md 或 rules 里加一句"公众号排版时读取 wechat-layout/SKILL.md 并按其规则执行"。这套规则不依赖任何 Claude 专有语法。

### 指定样式

```
用蒸汽波风格排版
用商务黑金主题、杂志编辑风排这篇年度总结
用我的品牌色 #ff6b35 生成一套主题再排版
```

不指定时模型按内容气质自动选。

### 粘贴进公众号

输出是 HTML 源码，直接贴源码不行。把它存成 `.html` 文件，浏览器打开，全选复制，再粘贴进公众号后台编辑器，样式就跟过去了。

## 公众号编辑器的坑

这几条都是实际踩过或从别人的方案里验证过的，写进了硬性约束：

- 样式必须全部内联，`<style>` 标签和 class 会被过滤（`rich_pages wxw-img` 等保留 class 除外）
- 直接包裹文字的最内层 `<span>` 要写成 `<span leaf="">`，否则粘贴后部分样式被剥离
- `position`、`transform`、`backdrop-filter`、`background-clip: text` 都不可用，毛玻璃效果只能用渐变加半透明边框模拟
- 图片要带 `class="rich_pages wxw-img"` 和 `max-width:100% !important`

## 层次和标注规则

前两版排出来的效果我自己不满意：两级标题长得几乎一样，正文大段没有抓手，扫一眼抓不到重点。后来改了两处，效果差别很大。

字号阶梯拉开。一级标题 22px 用卡片，二级 17px 只用左竖线，三级 15px 加粗带 ▍前缀，正文 15px，辅助文字 13px。相邻层级差至少 2px，标题上方留白是下方的 2 倍以上。自检标准是折叠正文只看标题，能还原出文章大纲。

双层标注，下限而非上限。高亮（色底）标核心结论，全文 5–10%；粗体（无底色）标术语、数据、动作词，超过 60 字的段落至少一处。单段超 120 字必须拆。自检标准是遮住普通文字，只看标题加标注能复述文章主线。

这两套自检写在 `SKILL.md` 的「视觉层次规范」里，模型输出前会过一遍。
