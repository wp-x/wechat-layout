# 风格实验室：从参考作品生成新风格的标准流程

用户给出参考文章/截图/网站，希望"做成这种感觉"时，按此流程执行。产出的新风格写入 `styles.md`，与既有风格同等地位。

## 第一步：五层解剖

对参考作品逐层提取，缺一层则风格会"形似神不似"：

| 层 | 提取什么 | 提问方式 |
|---|---|---|
| 1. 布局范式 | 内容的组织单位是什么 | 是"文章流"（标题+段落线性排布）还是"色带分区"（整页由全宽 section 交替组成，像官网）？还是卡片墙、杂志栅格？ |
| 2. 字体系统 | 字号阶梯、字重、字距、对齐 | 最大字号出现在哪、和正文差几倍？负字距还是宽字距？两端对齐还是左对齐？ |
| 3. 色彩系统 | 底色层次、主色用法、点缀色数量 | 背景是纯白还是有色底？主色用于文字、底色还是描边？有没有多点缀色（如产品官网常用 2-3 个撞色圆点）？ |
| 4. 组件形态 | 圆角、阴影、边框、图片处理 | 圆角多大？阴影是硬投影、弥散影还是无影？图片带不带圆角和图注？ |
| 5. 装饰语言 | 反复出现的小元素 | 大写英文 eyebrow？超大章节数字？内联 SVG 图标？胶囊徽标？分隔线造型？ |

统计手段：对 HTML 用 grep 统计 `font-size`、`background-color`、`border-radius` 的频次分布，高频值就是该风格的令牌。

## 第二步：抽象成令牌，而非复刻数值

把解剖结果改写成与配色无关的规则。参考里的 `#0098f2` 不写死，写成 `{{主题色}}`；只有当某颜色是风格 DNA（如蒸汽波的粉紫青）才允许专属色板。

判断标准：换一套主题色后该风格是否依然成立？成立的部分是"风格"，不成立的部分是"配色"，拆开。

## 第三步：公众号兼容性折算

参考作品往往来自网页，逐项检查并给出替代：

- `position/transform/动画` → 删，用静态布局重构
- `backdrop-filter` → 渐变 + 半透明边框模拟
- 自定义字体 → 系统字体栈里选最接近的
- 内联 SVG **可用**（`<svg>` 直接写在 HTML 里公众号不过滤），可承担图标与简单插画
- CSS Grid/Flex → 慎用，用 `display: inline-block` 或色带堆叠替代

## 第四步：发散变体（必做，至少 2 个）

复刻只是起点。抽象出布局范式后，问三个问题各生成一个变体方向：

1. **换情绪**：同一范式，亮色改深色、克制改张扬，适配什么新内容类型？
2. **换密度**：留白翻倍或砍半，变成"大气发布会"或"信息密集手册"？
3. **换血统**：把这个范式和已有 9 种风格里的某一种杂交（如色带分区 × 国风配色）会得到什么？

变体不必全部实装，在新风格的「变体」小节里记录规则差异即可，用户点名时再展开。

## 第五步：质量闸门

新风格入库前过三关：

1. **层次红线**：相邻标题层级字号差 ≥ 4px；三级灰度制；标题上方间距 ≥ 下方 2 倍。风格可以自定义整套字号阶梯（尤其是色带分区范式），但相对关系必须成立。
2. **扫读测试**：用该风格排一段 500 字样文，遮住正文只看标题和标注能否复述主线。
3. **粘贴测试**：生成的 HTML 存文件、浏览器打开、复制进公众号编辑器草稿，确认样式不丢（重点检查 `<span leaf="">`、SVG、深色底）。
